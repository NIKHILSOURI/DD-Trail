# NEVER install this file into the main PyTorch `venv`.
#
# TensorFlow 2.13 forces typing_extensions<4.6 (e.g. 4.5.0). PyTorch 2.7+ requires
# typing_extensions>=4.10 (TypeIs) — you will get: ImportError: cannot import name 'TypeIs'.
# TF also downgrades numpy; requirements-thoughtviz-gpu.txt can replace nvidia-cudnn-cu11
# and break the torch CUDA stack.
#
# If you already polluted the main venv:  bash scripts/repair_pytorch_venv.sh
#
# Use a SEPARATE venv from the main DreamDiffusion PyTorch stack, e.g.:
#   python3 -m venv venv_thoughtviz
#   source venv_thoughtviz/bin/activate
#   pip install -r requirements-thoughtviz.txt
#
# IMPORTANT — do NOT run `pip install -e ./code` in this venv. That installs the
# `dreamdiffusion` package, which pulls torch/timm/tqdm and will conflict with
# TensorFlow and trigger pip warnings like:
#   dreamdiffusion 0.0.1 requires timm/torch/tqdm, which is not installed
# If you already did that here, remove it:
#   pip uninstall dreamdiffusion -y
#
# Python: 3.10 or 3.11 recommended (TensorFlow 2.13 wheels; avoid mixing with
# a PyTorch env that already pins numpy/torch in conflicting ways).
#
# These pins match the legacy imports in code/ThoughtViz (standalone `keras`,
# keras.engine.topology / convolutional layers, Adam(lr=...), etc.).
